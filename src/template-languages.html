Eleventy and Template Languages

File Extensions

Eleventy can process around ten different template languages (See Intro for the list). Eleventy defaults to Liquid for pages with an HTML extension. Eleventy relies on the file extension to know what language is to be used, so to tell it to use Nunjucks, use the extension *.njk. This can also affect certain defaults, so it's useful just to indicate what language is intended, so even when using Liquid, the extension *.liquid can be used.

Change these filenames to the *.liquid extension. 

animals.html
fishpaged.html
index.html
postlist.html

_includes/foot.html
_includes/head.html
_includes/layout.html

Changing these filenames will affect includes and layouts, so some content will need to changed. For includes, the ".html" extension part can now be removed, since the template will default to ".liquid". And front matter pointing to layouts needs extensions to be changed to ".liquid". 

animals.liquid (include head and foot tags can have the ".html" removed)
fishpaged.liquid (include head and foot tags can have the ".html" removed)
index.liquid (include head and foot tags can have the ".html" removed)
about.md (change the extensions in the front matter to ".liquid")

posts/post-01.md (change the extensions in the front matter to ".liquid")
posts/post-02.md (change the extensions in the front matter to ".liquid")
posts/post-03.md (change the extensions in the front matter to ".liquid")

_includes/layout.liquid (change the extensions in the front matter to ".liquid")
Confirm everything processed correctly. Basically, everything in _site should look the same as before. Note how, even though the templates are now *.liquid, the output files are still *.html.

Liquid and Nunjucks

Different template languages have different advantages. Liquid is simple to use. However, a language like Nunjucks (which seems more popular) allows using JavaScript in front matter, for example. Create the file jsontest.njk. Note how the front matter starts with ---js. Note also the slightly different syntax in the template where there is a {{ loop.index }} tag instead of the {{ forloop.index }} tag used in Liquid. 

---js
{
    title: "JSON test page",
    body: "body text from JSON",
    arctics : function() {
        var arctics = ["walrus", "penguin", "puffin", "polar bear"];
        return arctics.sort();
    }
}
---
<!DOCTYPE html>
<html>
<head>
     <title>{{ title }}</title>>
</head>
<body>
    <h1>{{ title }}</h1>
    {{ body }}
    {% for arctic in arctics() -%}
        {{ loop.index }}. {{ arctic }}
    {% endfor -%}
</body>
</html>
A new folder and file _site/jsontest/index.html should be generated : 
<!DOCTYPE html>
<html>
<head>
    <title>JSON test page</title>
</head>
<body>
    <h1>JSON test page</h1>
    body text from JSON
    1. penguin
    2. polar bear
    3. puffin
    4. walrus
    </body>
</html>
For more details on what different template languages can do, check out the Eleventy documentation on languages.
