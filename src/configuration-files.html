Configuration Files

Target Path

Eleventy can be customized to match the way you prefer to work. For example, you can change the target folder from _site to dist. Configuration is done by creating the .eleventy.js file (note the dot at the beginning) : 

module.exports = {
    dir: {
        output: "dist"
    }
};
Delete the _site folder, re-run Eleventy. A new dist folder should appear.

Source Path

Instead of placing templates in the project root, Eleventy can process from a source folder, e.g. src. Edit .eleventy.js : 

module.exports = {
    dir: {
        output: "dist",
        input: "src"
    }
};
Move the *.liquid, *.njk, and *.md files, and posts, _includes and _data folders all into src. Delete the target folder, re-run Eleventy, and dist should re-appear with the same structure as before.

Template Formats

There should currently be both *.liquid and *.njk templates in the eleventy-test project folder. To tell Eleventy to only process *.liquid templates, edit .eleventy.js : 

module.exports = {
  
    dir: {
        output: "dist"
        input: "src"
    },

    templateFormats: ["liquid"]

};
Delete the dist folder, re-run Eleventy, and in the new dist folder there should no longer be a jsontest folder.

Add "njk" to the templateFormats : 

module.exports = {
  
    dir: {
        output: "dist",
        input: "src"
    },

    templateFormats: ["liquid", "njk"]

};
The jsontest folder should re-appear inside dist.
